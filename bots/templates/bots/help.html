Main menu: (only administrator sees all menus.)

  • Home:
    • Login.
    • Logout.
  • Last run:
    • Reports of all runs.
    • Incoming EDI files of last run.
    • Outgoing outgoing EDI files of last run.
    • Process errors of last run.
  • Select:
    • Reports using selection criteria.
    • Incoming using selection criteria.
    • Outgoing using selection criteria.
    • Process errors: using selection criteria.
    • Confirmations: view if confirmations are received or generated.
  • Maintainance: set up configuration.
    • Routes: where to get edi files, how to translate, where to put results.
    • Channels: configure comunication channels in and out.
    • Translations: translate what to what using mapping script.
    • Partners: maintain partners.
    • EmailAddresses: maintain channel specific email addresses.
    • Partnergroups: partners can be in a partnergroup.
    • Codelists: maintain codelists used in translations.
    • Confirmations rules: when should a confirmation be asked or generated.
  • System Tasks: administrator tasks.
    • Users: maintain bots system users.
    • Read a plugin: load a predefined edi configuration from a plugin.
    • Make snapshot plugin: generate a plugin of current configuration and data.
    • Make configuration plugin: generate a plugin of current configuration.
    • Delete all transactions.
    • Delete all configuration.
    • Delete all user maintained codes.
    • View/edit counters: for automatic counters in edi messages.
  • Run: receive, translate and send EDI messages.
    • Run only new: receive new EDI files only.
    • Run only retry: retry files with errors, eg communication errors.
    • Run only retransmit: retransmit/rerecieve edi files marked by user.
    • Run all: run new, retries and retransmits.
